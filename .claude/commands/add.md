# /add — เพิ่มกิจกรรมใหม่จากการ์ด

ผู้ใช้วางรูปการ์ด `.jpg` ไว้ใน `inbox/` แล้วเรียก `/add`

อ่าน `CLAUDE.md` ก่อนทำงาน ถ้ายังไม่ได้อ่านในเซสชั่นนี้

## ขั้นตอน

### 1. หาไฟล์การ์ด
- ดูว่าใน `inbox/` มีไฟล์อะไรบ้าง (ยกเว้น `.gitkeep`)
- ถ้าไม่มี → บอก user ว่า inbox ว่าง ขอให้วางการ์ดก่อน
- ถ้ามีหลายไฟล์ → ถาม user ว่าเป็นกิจกรรมเดียวกันหรือคนละกิจกรรม

### 2. อ่านรูปด้วย Vision
ใช้ Read tool บนไฟล์รูป จากนั้นถอดข้อมูล:
- `temple` — ชื่อวัด (ภาษาไทยเต็ม)
- `title` — ชื่องาน/พิธี
- `date` — วันที่ ถ้าในการ์ดเขียน พ.ศ. → แปลงเป็น ค.ศ. (ลบ 543) แล้ว format เป็น `YYYY-MM-DD`
- `contact` — เบอร์โทร/อีเมล/line ถ้ามี (ว่างไว้ถ้าไม่มี)

### 3. เลือก categories (เลือกได้หลาย)

Whitelist เท่านั้น: `["งานบุญ", "ทำบุญ", "เทศกาล", "งานอบรม", "ต่างประเทศ", "ศูนย์สาขา"]`

Heuristics:

| เจอคำ | ใส่ tag |
|---|---|
| ทอดผ้าป่า, ทอดกฐิน, บวช, สังฆทาน, ถวาย | `ทำบุญ` + `งานบุญ` |
| วิสาขบูชา, มาฆบูชา, อาสาฬหบูชา, เข้าพรรษา, ออกพรรษา, ลอยกระทง, สงกรานต์ | `เทศกาล` (+ tag ที่เกี่ยวข้อง) |
| อบรม, ปฏิบัติธรรม, ค่าย, ฝึกสมาธิ, retreat | `งานอบรม` |
| ชื่อประเทศที่ไม่ใช่ไทย, English name, ต่างประเทศ | `ต่างประเทศ` |
| ศูนย์สาขา, สำนักสาขา, สาขาที่... | `ศูนย์สาขา` |

`categories[0]` = primary tag (เด่นที่สุดในการ์ด — แสดงบน card)

### 4. เดา important

`important = true` ถ้าเจอคำเหล่านี้:
- วิสาขบูชา
- มาฆบูชา
- อาสาฬหบูชา
- เข้าพรรษา / วันเข้าพรรษา
- ออกพรรษา / วันออกพรรษา
- กฐิน / ทอดกฐิน
- วันพระใหญ่

ไม่เจอ → `false`

### 5. แสดง Preview

แสดง JSON แบบจัดสวย พร้อมเหตุผลที่เลือกแต่ละ tag:

```
📋 Preview กิจกรรมใหม่:

{
  "temple": "วัดพระธรรมกาย",
  "title": "พิธีทอดผ้าป่าวันวิสาขบูชา",
  "date": "2026-05-22",
  "categories": ["ทำบุญ", "เทศกาล", "งานบุญ"],
  "important": true,
  "contact": "02-395-0098"
}

หมวดที่เลือก:
  ✓ ทำบุญ (เพราะมีคำว่า "ทอดผ้าป่า")
  ✓ เทศกาล (เพราะตรงกับวันวิสาขบูชา)
  ✓ งานบุญ (default สำหรับกิจกรรมทำบุญ)
สำคัญ: true (เพราะวันวิสาขบูชา)

รูปที่จะใช้: inbox/<filename>.jpg
```

### 6. หยุดถาม

ห้ามรันคำสั่งใดๆ ต่อจนกว่า user จะตอบ — เลือก:
- `yes` / `y` / `ใช่` / `ok` → ไป step 7
- `edit` / `แก้` → ถามว่าจะแก้ field ไหน เปลี่ยนค่าตามที่ user บอก แล้วกลับไป step 5
- `cancel` / `ยกเลิก` / `no` → หยุด ไม่ทำอะไรกับ inbox

**ห้าม commit / push / รัน script ก่อนได้ "yes" จาก user เด็ดขาด**

### 7. รัน add_card.py

```bash
python scripts/add_card.py --json '<JSON>' --image inbox/<filename>.jpg
```

ถ้ามีหลายรูปสำหรับกิจกรรมเดียว → ใส่ `--image` หลายๆ ไฟล์

ถ้า exit code != 0 → แสดง error ให้ user เห็น **อย่ารัน deploy.py ต่อ**

### 8. รัน deploy.py

ถ้า add_card.py สำเร็จ → ทำต่อ:

```bash
python scripts/deploy.py
```

ถ้า deploy.py fail (เช่น .env ยังไม่ตั้ง / VPS ไม่ตอบ) → แจ้ง user แต่ commit ใน local + remote git ยังถือว่าสำเร็จแล้ว user สามารถ deploy เองทีหลัง

### 9. รายงานผล

ปิดท้ายด้วยสรุปสั้นๆ:
- ID ของ event ที่เพิ่ม
- ชื่อไฟล์รูปใน `cards/`
- commit hash
- URL เว็บ (ถ้า deploy สำเร็จ)

## ข้อห้ามสำคัญ

- ห้ามแก้ `events.json` ด้วยมือ — ใช้ `scripts/add_card.py` เท่านั้น
- ห้ามใส่ category นอก whitelist (จะ fail validation)
- ห้ามใส่ พ.ศ. ใน field `date` (ใช้ ค.ศ. YYYY-MM-DD)
- ห้ามใช้ git commands ตรงๆ — script จัดการให้
- ห้ามรัน `--no-git` หรือ `--dry-run` ถ้า user ตอบ `yes` (flags นี้สำหรับ dev เท่านั้น)
